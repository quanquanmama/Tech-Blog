# 1.快速搭建博客

1、安装

yarn global add vuepress #或者： npm install -g vuepress
1

2、 新建一个项目文件夹，并且初始化

mkdir myblog  #创建一个项目myblog
cd myblog   #进入这个目录
npm init -y #初始化项目
1
2
3

在package.json里面加入一些脚本：

{
    "script":{
        "dev":"vuepress dev docs",
        "build":"vuepress build docs"
    }
}
1
2
3
4
5
6

运行程序：

yarn dev # 或者：npm run dev
1

生成静态的HTML文件：

yarn build # 或者：npm run build
1

默认情况下，文件将会被生成在 .vuepress/dist，当然，你也可以通过 .vuepress/config.js中的 dest 字段来修改，生成的文件可以部署到任意的静态文件服务器上.

3、创建博客相关目录 在myblog目录下新建docs目录，在docs目录里新建.vuepress文件夹，然后在里面创建config.js文件 用于存放配置信息

4、在docs目录里面创建一个README.md的文件，输入内容

/docs/README.md

---
home: true
heroImage: /img/logo.jpg
heroText: Wiki 1001

  details: 活着，是一件最能带来满足感的事。细细琢磨吧，人生啊，有意思的很...

WARNING

图片等静态资源要放到public

此时，项目的目录结构为：

project
├─── docs
│   ├── README.md
│   └── .vuepress
│       ├── dist
│       ├── public
│       └── config.js
└── package.json
1
2
3
4
5
6
7
8

不管使用哪种方式，最终的项目结构应该和上面一样。

  • docs文件夹是你的根目录，也是vuepress要去解析的文件夹，
  • docs下的README.md可以理解为首页页面。
  • docs下的.vuepress是一些配置文件，这里可以存放图片等静态资源，一些主题配置，自定义组件等等

5、运行博客 在终端启动命令

npm run dev

# 2.config.js基本配置

js基本配置：

module.exports = {
    title: '米加蓝的技术博客 ', // 页签标题 # | Wiki 1001
    description: '欢迎来到我的博客', // meta 中的描述文字，意义不大，SEO用
// 注入到当前页面的 HTML <head> 中的标签
    head: [
        // 增加一个自定义的 favicon(网页标签的图标)
        // 这里的 '/' 指向 docs/.vuepress/public 文件目录 
        // 即 docs/.vuepress/public/img/geass-bg.ico
        ['link', { rel: 'icon', href: '/img/geass-bg.ico' }], 
    ],
    base: '/Tech-Blog/', // 这是部署到github相关的配置  Repository name 和这里对应 
    markdown: {
        lineNumbers: true // 代码块显示行号
    },
 }

config.js中继续配置主题参数 顶部导航栏

module.exports = {
    ...
    themeConfig: {
        sidebarDepth: 4, // 将同时提取markdown中h2 和 h3 标题，显示在侧边栏上。
        lastUpdated: 'Last Updated' ,// 文档更新时间：每个文件git最后提交的时间,
        // 顶部导航栏
        nav:[
             // 单项 text：显示文字，link：指向链接
             // 这里的'/' 指的是 docs文件夹路径
             // [以 '/' 结尾的默认指向该路径下README.md文件]
            { text: '求索', link: '/FAQ/' },  // http://localhost:8080/Wiki1001Pro/FAQ/
            { text: '仓库', link: '/Store/' },
            { text: '随笔', link: '/Thought/' },

                    { text: 'Segmentfault', link: 'https://segmentfault.com/u/mulander' },

            {
                text: 'GitHub',
                items: [
                    { text: 'GitHub首页', link: 'https://github.com/Mulander-J' },

# 4.挂载GitHub

  • 建立一个公有仓库 ： https://github.com/quanquanmama/Tech-Blog.git

  • 在 setting 里设置好github-page属性 config.js 中 base: '/Tech-Blog/', // 这是部署到github相关的配置 Repository name 和这里对应

  • 将build获得地dist文件夹上传至github 访问https://quanquanmama.github.io/Tech-Blog/

# 编写一个deploy.sh的脚本文件

在项目的根目录下新建一个deploy.sh的文件

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd dist

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io  填写你刚刚创建的仓库地址
git push -f https://github.com/quanquanmama/Tech-Blog.git master

cd -
